Understanding Alerts and Tickets

SIGHTING --> CID y, SID n, UID y (who seen with it, becomes owner), INFORM Y (current Owner)

USING --> CID y, SID n, UID n, INFORM n --> Closed on open

CLAIMING --> CID y, SID n, UID n, INFORM y (previous owner)

GIVING --> CID y, SID n, UID n, INFORM y (new owner)

BROKEN --> CID y, SID n, UID y (who will fix), INFORM y (current owner) -> NEEDS WORK LOG TO BE CLOSED!

LOST --> CID y, SID n, UID y (who will fix), INFORM Y (current owner) -> NEEDS WORK LOG TO BE CLOSED!

DIED --> CID y, SID n, UID y (who will fix), INFORM Y (current owner) -> NEEDS WORK LOG TO BE CLOSED!

CARE --> CID y, SID n, UID y (who will fix), INFORM Y (current owner) -> NEEDS WORK LOG TO BE CLOSED!

BACKUP --> CID y, SID n, UID n, INFORM n

INSTALL --> CID y, SID y, UID n, INFORM y (device current owner)

REMOVE --> CID y, SID y, UID n, INFORM y (device current owner)

REQUEST --> CID y, SID y, UID y, INFORM y (device current owner)
